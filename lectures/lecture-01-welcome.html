Introduction

Why This Course?

Your Digital Footprint

  • In the last 24 hours, you generated:
    • Dozens of social media interactions
    • Hundreds of GPS/location updates
    • Streaming/video activity logs
  • Multiply by billions of people → massive global data

From Data to Decisions

  • Data-driven decision making
    1. Collection → sensors, logs, surveys
    2. Processing → clean, organize, enrich
    3. Analysis → stats, ML, visualization
    4. Action → decisions, discoveries, policies
  • Processing turns raw data into something we can analyze.

Zoom In: Data Processing

  • Main tasks of data processing:
    • Cleaning → fix errors, handle missing values
    • Transforming → reformat, normalize, standardize
    • Enriching → add metadata, join data
    • Aggregating → group, summarize, compute statistics
  • Good processing makes data reliable for analysis.

Example Python Program

print("Hello, world!")
x = 5
y = 2 * x
print("y = ", y)
  • What does this program do?
    1. Print Hello, world! to the console
    2. Bind the value 5 to the variable x
    3. Compute 2 * x and bind the result to y
    4. Print y = followed by the value of y
  • Live demo: Script execution, Shell, Notebook

PYTHON PROGRAMS

  • A program = definitions + commands
  • Definitions → create values & names
  • Commands → tell Python what to do
  • Run code in a shell or a file
x = 5  # definition
print(x)  # command

Objects

  • Programs manipulate data objects
  • Objects have a type that defines its actions
    • A human can walk, speak, etc.
    • A dog can bark, fetch, etc.
  • Two categories of objects:
    • Scalar – cannot be subdivided
    • Non-scalar – have internal structure that is accessible
      • A string can be concatenated, split
      • A list can be appended, sorted

Scalar Objects

  • int – represent integers, e.g. 5
  • float – represent real numbers, e.g. 3.27
  • bool – represent Boolean values True / False
  • NoneType – special, has one value None
  • Use type() to see the type of an object
>>> type(5)
int
>>> type(3.0)
float

Type Conversions (Cast)

  • Converts object of one type to another
    • float(3) → converts integer 3 to float 3.0
    • int(3.9) → truncates float 3.9 to integer 3 
>>> float(3)
3.0
>>> int(3.9)
3

Printing to Console

  • Use print to show output
  • In Notebook, results appear below the code cell:
    • "Out" = result of expression (not printed) 
      In [11]: 3+2
Out[11]: 5
    • print = explicit program output 
      In [12]: print(3+2)
5

Expressions

  • Combine objects & operators to form expressions
  • An expression has a value, which has a type
  • Syntax for a simple expression:
    <object> <operator> <object>
  • Example:
    • the expression 3 + 5 has value 8 of type int 
    >>> type(3 + 5)
int

SIMPLE OPERATIONS

  • Parentheses tell Python what to do first
  • Operator precedence (without parentheses):
    • **
    • *
    • /
    • + and - executed left to right 
    • >>> 2 + 3 * 4
14
>>> (2 + 3) * 4
20

BINDING VARIABLES AND VALUES

  • The equal sign (=) is an assignment of a value to a variable name 
pi = 3.14159  # pi is a name, 3.14159 is a value
pi_approx = 22/7  # pi_approx is a name, 22/7 is a value
  • Value is stored in memory
  • Assignment binds name to value
  • Retrieve a value by typing its name: pi

ABSTRACTING EXPRESSIONS

Why give names to values of expressions?

  • To reuse names instead of values
  • Easier to change code later 
pi = 3.14159
radius = 2.2
area = pi * (radius ** 2)

PROGRAMMING vs MATH

  • In programming, you do not "solve for x" 
pi = 3.14159
radius = 2.2
area = pi * (radius ** 2)  # area of circle
radius = radius + 1
  • Assignment = store a value in a variable
    • Right side → expression evaluated to a value
    • Left side → variable name
    • radius = radius + 1 is equivalent to radius += 1

CHANGING BINDINGS

  • We can re-bind a name with a new assignment
  • Old value may still exist in memory; the name just points elsewhere
  • Computed values don't update automatically 
pi = 3.14
radius = 2.2
area = pi * (radius ** 2)
radius = radius + 1
  • Values of variables at each step:
    1. pi → 3.14
    2. radius → 2.2
    3. area → 15.1976
    4. radius → 3.2
